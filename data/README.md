# Data Directory

This directory holds the Bugs4Q benchmark data used by the pipeline.

## Directory structure (after preparation)

```
data/
└── bugs4q/
    └── bugs4q.json     ← one JSON array of program records
```

## What is Bugs4Q?

Bugs4Q is a benchmark of real bugs from Qiskit quantum-computing programs,
introduced in:

> Zhao et al., "Bugs4Q: A Benchmark of Real Bugs for Quantum Programs",  
> ASE 2021.  
> GitHub: https://github.com/Z-928/Bugs4Q

## How to prepare the data

### Option A – Synthetic data (quick start, no download needed)

Run the preparation script with no arguments:

```bash
python scripts/prepare_bugs4q.py
```

This generates 20 synthetic Qiskit program records in `data/bugs4q/bugs4q.json`
so you can immediately test the full pipeline without any data downloads.

### Option B – Real Bugs4Q data

1. Clone or download the Bugs4Q repository:

   ```bash
   git clone https://github.com/Z-928/Bugs4Q /tmp/bugs4q_repo
   ```

2. Run the preparation script with the `--source` flag pointing at the
   downloaded data:

   ```bash
   python scripts/prepare_bugs4q.py --source /tmp/bugs4q_repo
   ```

   The script converts the data into the expected JSON format and writes it
   to `data/bugs4q/`.

3. Verify the output:

   ```bash
   python -c "from src.dataset_loader import load_bugs4q; print(len(load_bugs4q()), 'records loaded')"
   ```

## JSON record format

Each record in `bugs4q.json` must contain at least:

| Field         | Type           | Description                                     |
|---------------|----------------|-------------------------------------------------|
| `id`          | `str`          | Unique program identifier                        |
| `source_code` | `str`          | Raw Python/Qiskit source code                    |
| `has_bug`     | `bool`         | Ground-truth: `true` if the program has a bug   |
| `bug_class`   | `str \| null`  | Taxonomy label (see `knowledge_base/taxonomy.json`) |
| `location`    | `str \| null`  | `"file:line"` hint, if known                    |
| `description` | `str \| null`  | Optional natural-language description            |

## Notes

- **Do not commit** raw Bugs4Q data files to this repository.
  The `data/bugs4q/` directory is listed in `.gitignore`.
- Synthetic records generated by `prepare_bugs4q.py` are safe to commit
  if you want to share a reproducible small dataset.
